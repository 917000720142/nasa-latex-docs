Quick Start Guide¶

The fastest and most efficient way to start a new document is utilize the buildPDF.py script’s --structure option in order to create a document tree with the minimum necessary files. For example, let’s assume you would like to create a new document whose main (root) .tex is to be titled XYZ_Conference_Paper.tex (it does not currently exist) and you want all the document content to be housed in a directory named XYZ_Conference_Paper_2017/. All of this may be accomplished through the following:

./buildPDF.py XYZ_Conference_Paper.tex -s XYZ_Conference_Paper_2017

This will create a document with the same structure as described in the general document architecture section as well as some sample files that can be used as starting points for expanding the document and placeholder Lorem Ipsum text.

The Root TeX File¶

Once the new document folder is created, it will contain your XYZ_Conference_Paper.tex file with the following contents:

% Define Class: %%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
\documentclass[]{nasa-latex-docs} 

% Preamble Section: %%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%


% Add bibliogrpahy file
\addbibresource{XYZ_Conference_Paper.bib}

% Main Document Content: %%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
\begin{document}

\input{tex/sample_content.tex}

\printbibliography

\end{document}
% End of Document %%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%

Currently, the document will be built with no template and in turn no pre-defined formatting. We can specify that we would like to create an AIAA conference paper by calling that aiaa-conference template as follows:

\documentclass[template=aiaa-conference]{nasa-latex-docs}

We can now build the entire document via:

./buildPDF.py XYZ_Conference_Paper.tex

From here more content can be added and the document can grow as required.

Declaring the Root Tex File¶

A neat feature of buildPDF.py is that it can pick up “magic” comments that are used by some other document compilers. Specifically, a magic comment can be placed at the top of each content file (located in the tex/ directory) so that the TeX root document can be explicitly defined. Suppose we create a new file within tex/ named introduction.tex with the following:

%!TEX root = ../XYZ_Conference_Paper.tex

\section{Introduction}

This is my introduction section.

We can either use buildPDF.py to build the root XYZ_Conference_Paper.tex file directly as described above, or we can build introduction.tex directly and buildPDF.py will know to point to the root document.

./buildPDF.py tex/introduction.tex

This is useful for any custom defined build shortcuts (example a keyboard shortcut to directly call buildPDF.py) that users may come up with. By placing the %!TEX root = comment at the top of each file it allows users to utilize the shortcuts to build the document without having to physically switch back to the root document.
