General Document Architecture¶

Note

Although this generic document architecture is not required to build a document, it is generally good practice to adhere to these structure principals for both clarity and ease of use when document complexity grows.

It is good practice to create a new folder (project directory) for each new LaTeX document. The folder structure for a project architecture has many possibilities, but nasa-latex-docs will present a basic one that can be generated using buildPDF.py (as described here).

This document structure is summarized below:

  • Project Directory (where all document content is housed)

    • root.tex
      • This is the root level .tex file that is used to build the document
      • Contains the \documentclass[]{} definition
      • Contains any document preamble content
      • Contains any nasa-latex-docs template document parameters
      • Contains the \begin{document} and \end{document} directives
      • Contains the \input{} or \include{} directives for all other document .tex file(s)
    • bib/ : Contains the bibliography file(s) for referencing
    • fig/ : Contains all figures and images used within the document
    • tex/ : Contains the actual document content in the form of separate .tex file(s)
    • Any other folders that the user finds useful for their own document architecture

Document Search Path Visibility¶

When using the buildPDF.py script, it is important to understand what is going on under the hood with regards to the user’s environment and where document content is searched for. When using the buildPDF.py script, the user’s environment is automatically configured to add the following to the search path environment variable, TEXINPUTS, that is used by LaTeX to search for content:

  • All folders and files under the nasa-latex-docs/ repository folder
  • All folders and files at or under the level of the root TeX file that is being built.
  • All folders and files that are passed to buildPDF.py using the --texinputs option

The --texinputs option to buildPDF.py is useful for end users to define a custom location on their computer where they can place TeX documents that are common to all of their documents. Examples of common files include a comprehensive bibliography list, an acronyms database, or even their own custom defined templates.
